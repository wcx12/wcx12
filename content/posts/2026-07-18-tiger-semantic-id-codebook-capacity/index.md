> English version: [When Recommenders Start to 'Write' Items: Why TIGER's Semantic IDs Collapse](../tiger-semantic-id-codebook-capacity-en/)

> **导语：** 从首层 256 个码字只剩 1 个的异常出发，我们在五个 Amazon 类目上追问：一个商品的离散“名字”应该怎样分层，才能既减少碰撞，又真正帮助推荐？

*一项覆盖五个数据集、四类容量排列与固定顺序消融的实证研究*

大多数推荐系统的工作方式可以粗略概括为“打分”：先取出一批候选商品，再给每个候选打分并排序。生成式推荐换了一个视角。它不再只问“候选 A 和候选 B 谁更合适”，而是像语言模型续写下一个词那样，**直接生成下一个商品的标识符**。

这立刻带来一个基础问题：商品原本使用的数据库 ID 只是随机编号，数字相近不代表内容相近，而且几百万个商品会形成巨大的输出词表。于是，生成式推荐需要一种更容易预测、又能保留商品语义的“名字”。

## TIGER 是什么

[TIGER](https://proceedings.neurips.cc/paper_files/paper/2023/hash/20dcab0f14046a5c6b02b61da9f13229-Abstract-Conference.html) 的全称是 **Transformer Index for GEnerative Recommenders**。这项发表于 NeurIPS 2023 的工作提出：先从商品标题、描述等文本中提取向量，再把向量量化成一串离散 token，最后训练 Transformer 根据用户历史生成目标商品的 token 序列。论文把这串 token 称为 **Semantic ID，也就是语义 ID**。

![TIGER 从商品文本到三层语义 ID，再到生成式推荐的五步流程](media/tiger-pipeline.png)

这里有三个容易混淆的词：

- **码本（codebook）**：某一层可选 token 的词典。
- **码字（codeword）**：码本中的一个具体条目，例如第一层的第 12 号条目。
- **语义 ID（Semantic ID）**：一个商品在所有层上的码字组合，例如 `(12, 87, 203)`。

TIGER 使用的量化器是 RQ-VAE。RQ 是 residual quantization，也就是**残差量化**：第一层先近似商品向量，第二层只编码第一层没有解释掉的误差，第三层再编码前两层剩余的误差。这种做法源自 [RQ-VAE 的 CVPR 2022 工作](https://openaccess.thecvf.com/content/CVPR2022/html/Lee_Autoregressive_Image_Generation_Using_Residual_Quantization_CVPR_2022_paper.html)。TIGER 使用 [Sentence-T5](https://aclanthology.org/2022.findings-acl.146/) 生成商品文本向量，再把 RQ-VAE 得到的离散序列用于推荐。

可以把三层语义 ID 想成地址。第一层决定大区域，第二层在区域内继续分叉，第三层找到更细的位置。这个比喻并不要求每层真的对应人工类目，但它准确描述了自回归预测的条件关系：模型必须先预测第一层，后两层才能在这个前缀下继续选择。

## 问题从哪里出现

如果三层码本各有 256 个码字，名义上可以组合出

$$
256^3=16{,}777{,}216
$$

个不同语义 ID。我们首先研究的 Amazon Reviews 2018 `Sports & Outdoors` 类目在预处理后有 91,651 个商品，表面上远小于 1,678 万。因此，直觉很容易给出一个错误答案：只要组合空间够大，商品就不会挤在一起。

实际训练却出现了相反的结果。未改动的官方配置给第一层准备了 256 个码字，训练结束后却只使用了 **1 个**。所有商品先经过同一个入口，只能依赖第二、第三层勉强区分。完整语义 ID 只有 `20.40%` 是唯一的，只有 `5.69%` 的商品独占一条路径。

这就是本文要拆开的核心矛盾：**名义容量很大，不代表训练后形成了足够多、足够均衡、也足够容易预测的有效路径。**

## 先说结论

1. **官方配置的浅层坍缩不是单个数据集的偶然。** 在 Sports、Toys、Office、Video 和 Musical 五个类目中，第一层的有效分支都远低于名义容量。
2. **“用过多少码字”不能替代推荐指标。** reset 可以显著提高码本利用率，却可能让 NDCG 更差；总熵、唯一 ID 比例和有效路径数也都不能单独选模型。
3. **hard-ST 修复的是训练目标与实际离散分桶之间的错位。** 它在五个数据集上同时缓解碰撞并提高下游推荐效果。
4. **容量放在哪一层，比容量乘积本身更重要。** 适度前置通常更稳，但 Musical 偏好后置，说明最佳层级结构由任务和数据共同决定。

下面先沿着实验主线解释这些结论是怎样得到的。公式、完整统计和结论边界放在后半部分的“技术详解”中。

## 从一个首层只剩 1 个码字的异常开始

### 码本利用率首先是故障信号，不是成绩单

第一层坍缩已经说明量化器出了问题，但它并没有让推荐模型完全失效。这个官方基线在 Sports 的验证集上仍取得 `0.2376` 的 NDCG@10，甚至比后来若干“码本利用率更漂亮”的方案更好。

NDCG@10 衡量正确商品是否出现在前 10 个结果中，并且会给排名更靠前的命中更高权重；数值越高越好。这个反常结果带来了第一条重要认识：

> **码本利用率是故障信号，不是推荐质量的替代指标。**

### reset 救活了码字，却没有救活推荐

我们先尝试了最自然的办法：让初始化更分散、鼓励更高熵，并定期重置长期不用的码字，这就是常见的 dead-code reset。离线的平衡残差 KMeans 证明，同一批商品完全可以把三层 256 个码字全部用起来，所以数据本身并没有“天然只能聚成一类”。

训练实验也确实让表面数字变好。平衡初始化加 dead-code reset 后，三层码本利用率提高到 `100%/71.5%/21.5%`，唯一语义 ID 比例提高到 `61.59%`。可是测试集 NDCG@10 只有 `0.1656`，仍低于官方基线的 `0.2122`。

这个结果排除了一个很诱人的叙事：**只要把闲置码字救活，推荐自然会变好。** 码字可以被使用，却仍可能形成高度冗余、不断漂移或不利于生成模型学习的路径。

### 三层不能只顾着各自“用满”

RQ-VAE 的后一层量化前一层留下的残差。如果三层同时快速更新，前层一变，后层看到的目标也跟着变。我们因此尝试逐层训练、降低旧层学习率、dead-code reset，以及允许已训练前缀继续适应新层的 active-prefix 方案。

这批实验里，真正有用的不是“分层训练”四个字，而是 active-prefix 的结果：它把唯一语义 ID 比例推到 `79.4%`，并把验证集/测试集 NDCG@10 提高到 `0.2749/0.2353`，第一次稳定超过官方基线。

随后我们把 decoder 从 10k 继续训练到 20k，恢复 optimizer state，结果没有出现更好的 checkpoint。这告诉我们，剩余差距不只是 decoder 少训练了一会儿；瓶颈仍在 SID 的学习目标和路径结构。

### soft 正则制造了一次“看起来很均匀”的假成功

为了避免复杂的 layerwise recipe，我们转而设计可导的分布正则：让每层 item 对 code 的平均 soft assignment 更接近均匀，并限制单个 code 的最大质量；随后又加入 prefix-conditioned 正则，希望每个已有前缀下面仍保留足够后继。

全局 soft 正则确实提高了下游：Sports 的验证集/测试集 NDCG@10 从官方基线的 `0.2376/0.2122` 提高到 `0.2633/0.2301`。较温和的前缀正则也进一步改善了完整路径熵、唯一 ID 比例和 singleton 比例。

但训练后期出现了一个矛盾：soft 分布的最大概率约为 `0.00399`，几乎等于 256 类均匀分布的 `1/256=0.003906`；真正用于生成 SID 的 hard argmin 却仍只用了 `75/256` 个第一层 code。

换句话说，正则看到的世界已经接近完美，部署时使用的离散世界仍然坍缩。这不是“权重还不够大”，而是优化对象错位。

### hard-ST 让训练时统计的分桶与实际使用的分桶一致

下一步修改很小，但逻辑关键：前向计算直接使用最近码字产生的离散 one-hot 分桶，反向传播仍沿连续的 soft 概率传递梯度。这种“前向是硬选择、反向保留软梯度”的 straight-through 估计，本文简称 **hard-ST**。

10k-step 短探针已经把三层码本利用率提高到 `98.4%/94.9%/96.9%`。完整 100k-step 等宽实验得到 `74.98%` 的唯一语义 ID，Sports 的验证集/测试集 NDCG@10 达到 `0.3228/0.2835`。

到这里，我们才第一次拥有一个同时满足三件事的基线：hard bucket 不再严重坍缩、碰撞显著减少、下游推荐也稳定提高。

这一步还改变了后续问题。此前我们一直在问“怎样让路径不要塌”；从这里开始，可以问一个更细的问题：**当路径基本可用时，容量到底应该放在哪一层？**

### 相同的 1,678 万条名义路径，容量放置方式仍会改变结果

我们固定三层容量乘积为 `2^24`，比较后置 `[128,256,512]`、等宽 `[256,256,256]`、中度前置 `[512,256,128]`、极端前置 `[1024,256,64]`，并在 Sports 上补齐另外两种排列。

Sports 的赢家是中度前置 `[512,256,128]`，valid/test NDCG@10 为 `0.3494/0.3070`。但极端前置虽然拥有更高的 H(full)、更多有效路径和更多 unique SID，NDCG 反而回落到 `0.3306/0.2905`。

这一步否定了另一个简单答案：**不是越多容量放在前面越好，也不是有效路径越多越好。** 第一层需要足够分辨率来做早期路由，中层还必须保留实质分叉能力，末层则承担剩余消歧。三者是一个联合配置问题。

### 固定同一批语义 ID，粗到细的预测顺序仍然更好

容量实验同时改变 RQ 表示和 decoder 输出头，仍然不够干净。于是我们固定同一个 `[512,256,128]` RQ-VAE、同一份 item-to-SID 映射，只把 decoder 预测顺序从 `L0 -> L1 -> L2` 改为 `L2 -> L1 -> L0`。

Sports 的 valid/test NDCG@10 相对下降 `7.89%/8.03%`。后来在 Toys、Office、Video、Musical 上复现，正序在五个数据集上全部获胜。

这条证据比跨容量比较更直接：同样三枚 token 并不是无序标签集合。早期 token 会成为后续预测的条件，粗到细的路径顺序本身具有任务价值。

### 五个数据集给出了稳定主线，也留下了一个反例

我们随后完成了 Sports、Toys、Office、Video、Musical 五个 Amazon 2018 类目的预注册矩阵。

中度前置相对等宽在五个数据集的 validation NDCG@10 上全部提高，但它只在 Sports、Toys、Office 三个数据集成为最终 winner。Video 选择极端前置，Musical 则选择后置。

与此同时，官方基线的第一层坍缩在五个数据集上全部出现：第一层有效分支只有 `1.00` 到 `6.19`。hard-ST 等宽方案在五个数据集上都提高了唯一语义 ID 比例和商品级 NDCG。

因此，我们可以比较有把握地说两件事：

1. 官方训练方案的浅层坍缩不是 Sports 的偶然特例。
2. 适度前置是一个稳健默认方向，但最优容量排列确实受任务和数据结构调节。

### Musical 把“总路径数”和“信息放在哪层”彻底分开

Musical 是整条主线里最有价值的反例。后置 `[128,256,512]` 的 valid NDCG@10 是 `0.3882`；极端前置 `[1024,256,64]` 只有 `0.3630`。但极端前置的有效完整路径更多，unique SID 也更高。

真正不同的是信息分配：

- 后置的条件有效分支是 `99.4/36.25/2.28`；
- 极端前置是 `970.3/6.56/1.34`。

两者的总 H(full) 几乎一样，极端前置却让第一层承担了绝大多数区分，第二、第三层快速退化为每个 prefix 下只剩一两个有效后继。后置方案保留了更强的逐层增量。

现有证据支持“条件有效分支衰减过快可能损害 Musical 的分层预测”，但还不能把它写成唯一原因。容量排列同时改变了量化几何、标签空间和 decoder 难度，下一步需要 oracle-prefix、固定 SID 的层损失干预和匹配数据量实验来做因果拆分。

### 最终得到的不是一个放之四海而皆准的配置

如果只想要一个当前默认值，中度前置 `[512,256,128]` 是五数据集平均排名最稳的候选。但这不是整项探索最重要的结论。

更重要的是，我们把一个模糊的“码本坍缩问题”拆成了四个不同问题：

1. **训练分配问题：** code 是否真正形成有质量的 hard bucket，而不只是 soft usage 好看。
2. **条件路径问题：** 每个 prefix 下还有多少有效后继，信息是在逐层增加还是很早耗尽。
3. **下游可学习性问题：** decoder 是否能可靠预测早期路由，并利用后续 token 细化。
4. **评价协议问题：** tuple 命中是否因为 SID 碰撞而产生乐观偏差。

所以当前最准确的阶段性结论是：

> **TIGER 的主要瓶颈不是名义组合空间不足，而是 hard assignment、条件路径形状与下游预测任务没有对齐。hard-ST 缓解了训练层面的浅层坍缩；适度前置和粗到细顺序通常更好，但最佳信息分配仍然由数据集决定。**

## 技术详解：先统一我们到底在测什么

### 为什么码本利用率、唯一语义 ID 和有效路径不是同一个量

从这里开始进入技术细节。不关心公式的读者可以只看每节开头和加粗结论；后续定义、推导与完整表格用于说明数字是怎样计算出来的。

假设商品集合共有 $N$ 个商品。第 $l$ 层有 $K_l$ 个码字，商品 $i$ 的三层语义 ID 写作

$$
S(i)=\bigl(S_0(i),S_1(i),S_2(i)\bigr).
$$

码本利用率（usage）只检查一个码字是否至少出现一次。即使一个码字容纳 90% 的商品、其余码字只分到极少商品，利用率仍可能接近 100%。因此还必须同时观察分布熵、完整路径碰撞和条件分叉。

#### 技术细节：边际熵、条件熵和有效分支怎样计算

对任意离散变量 $X$，用样本频率估计概率：

$$
\widehat p(x)=\frac{n_x}{N}.
$$

经验熵定义为

$$
H(X)=-\sum_x \widehat p(x)\log \widehat p(x).
$$

如果该层有 $K$ 个 code，归一化边际熵为

$$
\widetilde H(X)=\frac{H(X)}{\log K}.
$$

它等于 1 时表示经验分布完全均匀；接近 0 时表示质量集中在极少 code 上。

三层 SID 的熵满足链式法则：

$$
H(S_0,S_1,S_2)
=H(S_0)
+H(S_1\mid S_0)
+H(S_2\mid S_0,S_1).
$$

我们把每一项指数化，得到条件有效分支：

$$
\begin{aligned}
B_0&=\exp\bigl(H(S_0)\bigr),\\
B_1&=\exp\bigl(H(S_1\mid S_0)\bigr),\\
B_2&=\exp\bigl(H(S_2\mid S_0,S_1)\bigr).
\end{aligned}
$$

于是有效完整路径数为

$$
\begin{aligned}
N_{\mathrm{eff}}
&=\exp\bigl(H(S_0,S_1,S_2)\bigr)\\
&=\exp\bigl(H(S_0)\bigr)
  \exp\bigl(H(S_1\mid S_0)\bigr)
  \exp\bigl(H(S_2\mid S_0,S_1)\bigr)\\
&=B_0B_1B_2.
\end{aligned}
$$

这是熵链式法则的直接恒等变形，不是由实验结果拟合出来的经验公式。

### 为什么模型只能由验证集选择

所有生成模型都按静态验证集 NDCG@10 选择最佳 checkpoint。测试集、三类动态测试和 tuple-test 只在选择冻结后报告。这样做不是形式要求，而是为了避免在几十组量化器配置里无意中把测试集当成调参集。

#### 评价协议：商品候选重排与 tuple 生成有什么区别

主指标使用固定候选重排：每个样本包含 1 个目标商品和 99 个负样本，生成模型对候选商品的语义 ID 路径计分，再计算 HR/NDCG。

tuple 生成不给候选商品，只在合法的语义 ID 前缀树中生成 top-10 tuple，然后检查目标 tuple 是否命中。若多个商品共用同一个完整语义 ID，一次 tuple 命中会同时覆盖这些商品。因此碰撞严重的模型可能出现“商品排序更差、tuple NDCG 反而更高”。

这也是为什么本文从不使用 tuple-test 选择配置。

## 官方基线到底塌成了什么样

Sports 官方基线使用三层 `[256,256,256]` 码本、KMeans 初始化与 straight-through 梯度估计，RQ-VAE 训练 400k step，生成模型训练 10k step。它是后续所有修改的固定参照，而不是我们根据当前结果重新挑选出来的弱基线。

#### 查看完整数据：Sports 官方基线

| 指标 | 结果 |
|---|---:|
| RQ eval total / reconstruction / rq | `0.2258 / 0.1422 / 0.0836` |
| usage L0/L1/L2 | `1/256, 201/256, 179/256` |
| 条件有效分支 $B_0/B_1/B_2$ | `1.00 / 184.31 / 61.16` |
| H(full) / 有效完整路径 | `9.330 / 11,273` |
| unique / singleton | `20.40% / 5.69%` |
| valid HR@10 / NDCG@10 | `0.3980 / 0.2376` |
| test HR@10 / NDCG@10 | `0.3671 / 0.2122` |

第一层只有一个有效分支，但第二、第三层仍分别提供约 184 和 61 个条件有效分支，所以模型没有完全失去区分能力。也正因为如此，“发现 L0 坍缩”不能直接等价为“官方下游一定最差”。

## 为什么初始化和 reset 救不回下游推荐

这一阶段的核心价值不是找到最终方法，而是排除错误解释。balanced KMeans 证明 embedding 几何允许高覆盖；reset 证明 dead code 确实是问题的一部分；但推荐结果证明它们都不是充分条件。

#### 查看完整数据：初始化、探索和 reset

| 方法 | 修改动机 | RQ loss | usage L0/L1/L2 | H(full) | unique / singleton | valid NDCG | test NDCG |
|---|---|---:|---:|---:|---:|---:|---:|
| high-entropy probe | normalize、rotation trick、KMeans init 等增强探索 | `0.1416` | `57.8%/34.4%/19.1%` | `9.596` | `39.16%/24.0%` | `0.1164` | `0.1013` |
| balanced RQKMeans init | 均衡 residual 聚类初始化 | `0.1340` | `85.2%/38.3%/11.7%` | `10.151` | `48.91%/31.7%` | 未训练 | 未训练 |
| balanced init + reset | 定期重置长期零计数 code | `0.1322` | `100%/71.5%/21.5%` | `10.488` | `61.59%/46.5%` | `0.1900` | `0.1656` |

离线 balanced residual KMeans 在 raw-768 和 encoder-32 两种空间都能形成三层 100% usage。初始化时点还达到 `94.35%` unique 和 `89.92%` singleton，最大完整桶仅 24。训练后重新集中，因此主要矛盾在训练动力学，而不是数据无法被分散量化。

#### 结论边界：这一阶段排除了哪些解释

1. **不是 embedding 天然只有一个簇。** 离线均衡聚类可以覆盖全部 code。
2. **不是 dead code 唯一造成坍缩。** reset 提高了 usage，却没有恢复下游。
3. **不是 reconstruction 越低越好。** 早期探索的 reconstruction 改善没有转化成 NDCG。
4. **不是 unique 越高越好。** balanced+reset 的 unique 明显高于官方，下游仍低于官方。

## 逐层训练的启示：前缀不能只被冻结，也不能无限漂移

layerwise lr decay 的初衷是让后层面对更稳定的 residual；但旧层过早冻结或只保留很小学习率时，前层自身会退化。active-prefix 的关键变化是：训练新层时，已经激活的前缀层仍能共同适应。

#### 查看完整数据：逐层训练系列

| 方法 | 核心变化 | usage L0/L1/L2 | H(full) | unique / singleton | valid NDCG | test NDCG |
|---|---|---:|---:|---:|---:|---:|
| layerwise lr decay | 每层约 33k step，旧层低学习率 | `33.2%/100%/-` | `7.008`，仅两层 | `6.65%/2.1%` | 未训练 | 未训练 |
| layerwise + reset | 分层训练并修复 dead code | `78.5%/100%/100%` | `9.982` | `50.59%/36.1%` | `0.1433` | `0.1212` |
| active-prefix + reset 60k | 旧层随新层继续更新 | `86.7%/100%/100%` | `10.977` | `79.4%/69.7%` | `0.2749` | `0.2353` |

layerwise+reset 的后两层 usage 都是 100%，下游却是这组里最差之一。这再次说明“每层都用满”并不代表完整路径均衡，也不代表 decoder 能学。

#### 控制实验：延长生成模型训练为什么没有改变结论

我们固定同一个 active-prefix RQ-VAE，从原 10k decoder best checkpoint 继续训练，并恢复 optimizer state。20k 内没有任何新 checkpoint 超过原 best，最终 valid/test 仍是 `0.2749/0.2353`。

这个实验只排除一个窄解释：active-prefix 与更强方法之间的差距，不是单纯因为 decoder 少跑 10k step。它不能证明 decoder 架构已经最优。

## 为什么 soft 正则会“看起来已经均匀”

soft regularizer 的目标是用可导概率近似真实 assignment。它在工程上有效，也带来真实 NDCG 增益；问题在于真实 SID 由 hard argmin 决定，而软概率可以在边界附近看起来均匀。

#### 技术细节：soft 分布正则的精确定义

对 item $i$ 和第 $l$ 层 code $k$，先计算归一化距离 $\widetilde d_{ik}^{(l)}$，再定义温度为 $\tau$ 的 soft assignment：

$$
q_{ik}^{(l)}
=\frac{\exp\left(-\widetilde d_{ik}^{(l)}/\tau\right)}
{\sum_{j=1}^{K_l}\exp\left(-\widetilde d_{ij}^{(l)}/\tau\right)}.
$$

batch 平均 usage 为

$$
\overline q_k^{(l)}=\frac{1}{B}\sum_{i=1}^{B}q_{ik}^{(l)}.
$$

实现中的熵缺口是

$$
\Delta_H^{(l)}
=\max\left(0,\log K_l-H\bigl(\overline q^{(l)}\bigr)\right),
$$

最大质量惩罚为

$$
P_{\max}^{(l)}
=\left[\max_k\overline q_k^{(l)}-\rho_l\right]_+^2.
$$

单层正则为

$$
\mathcal L_{\mathrm{dist}}^{(l)}
=\lambda_H\Delta_H^{(l)}+\lambda_M P_{\max}^{(l)}.
$$

代码对 row-std normalization 的定义是：先减去每个 item 的最小距离，再除以该 item 对全部 code 距离的标准差。它解决的是距离尺度问题，不改变 hard argmin。

#### 技术细节：500-step 数值尺度探针

| 探针 | eval total | usage L0/L1/L2 | 观察 |
|---|---:|---:|---|
| baseline 500 | `0.254` | `0.4%/0.8%/0.8%` | 官方早期快速集中 |
| raw `tau=3e-7,h=.02,m=1` | `3.541` | `0.4%/0.8%/0.8%` | 正则过大，无探索收益 |
| raw `tau=1e-6,h=.05,m=10` | `30.620` | `0.4%/0.4%/0.4%` | softmax 饱和 |
| raw `tau=1e-6,h=.1,m=30` | `83.154` | `0.4%/0.8%/0.8%` | 正则主导训练 |
| row-std `tau=.05,h=.01,m=3` | `0.261` | `0.8%/0.8%/0.8%` | 稳定但梯度偏弱 |
| row-std `tau=.05,h=.02,m=10` | `0.293` | `2.3%/2.0%/1.6%` | 有少量探索 |
| row-std `tau=.02,h=.01,m=3` | `0.284` | `0.8%/2.3%/3.9%` | 选作温和完整配置 |

这些探针只用于找数值尺度，没有 decoder，不能作为推荐结果比较。

#### 查看完整数据：全局正则、前缀正则与 soft/hard 错位

global soft 100k 把 Sports L0 usage 从 `0.4%` 提高到 `24.6%`，H(full) 从 `9.330` 提高到 `10.683`，valid/test NDCG 从 `0.2376/0.2122` 提高到 `0.2633/0.2301`。

prefix 正则按已有 prefix 分组。对一个大小为 $n_g$ 的 prefix group，目标容量取

$$
K_g^*=\min(K_l,n_g),
$$

目标熵为 $\log K_g^*$，最大质量阈值为 $\rho_{\mathrm{factor}}/K_g^*$。这样不会要求一个只有 5 个 item 的 prefix 均匀占满 256 个 code。

10k 探针结果：

| 配置 | L0 usage | H(full) | unique | singleton | 最大完整桶占比 |
|---|---:|---:|---:|---:|---:|
| 仅 global soft | `12.9%` | `10.497` | `56.52%` | `40.07%` | `0.0884%` |
| gentle prefix | `18.0%` | `10.585` | `60.00%` | `44.01%` | `0.0807%` |
| stronger prefix | `9.0%` | `10.182` | `46.70%` | `30.08%` | `0.1015%` |

gentle 完整训练得到 L0 usage `29.3%`、H(full) `10.766`、unique/singleton `66.5%/51.2%`。但 soft 最大概率约 `0.00399` 时，hard L0 仍只有 `75/256` 个 code。这个矛盾直接触发 hard-ST。

## hard-ST：让正则真正看到实际使用的离散分桶

hard-ST 的核心不是把 entropy weight 继续增大，而是改变正则前向计算使用的 assignment。训练时的统计量与最终 SID 都看同一个 hard argmin，梯度则通过 soft probability 近似传播。

#### 技术细节：hard-ST 的前向与反向

令 soft assignment 为 $q_{ik}$，hard argmin one-hot 为

$$
h_{ik}=\mathbf 1\left[k=\arg\min_j d_{ij}\right].
$$

记 $\operatorname{sg}(x)$ 为 stop-gradient：前向值等于 $x$，反向导数为 0。hard-ST assignment 定义为

$$
\widetilde q_{ik}
=h_{ik}+q_{ik}-\operatorname{sg}(q_{ik}).
$$

前向时 $q_{ik}-\operatorname{sg}(q_{ik})=0$，所以

$$
\widetilde q_{ik}=h_{ik}.
$$

反向时 $h_{ik}$ 和 $\operatorname{sg}(q_{ik})$ 都不提供梯度，因此

$$
\frac{\partial \widetilde q_{ik}}{\partial q_{ik}}=1.
$$

这正是 hard-forward / soft-backward：正则损失前向统计真实 hard bucket，反向仍能更新距离和 codebook。它是一个梯度估计器，不是 hard argmin 的真实导数。

#### 查看完整数据：hard-ST 短探针与完整训练

10k 探针已经达到：usage `98.4%/94.9%/96.9%`、H(full) `10.939`、unique `74.11%`、singleton `60.79%`、最大完整桶 38。

完整 100k 等宽结果：

| 指标 | hard-ST `[256,256,256]` |
|---|---:|
| RQ total / reconstruction / rq / distribution | `0.2883 / 0.1128 / 0.1632 / 0.0122` |
| usage | `252/256, 255/256, 255/256` |
| H(full) / 有效完整路径 | `10.9529 / 57,119` |
| unique / singleton | `74.98% / 62.34%` |
| 最大完整桶 | `53` |
| valid HR@10 / NDCG@10 | `0.5074 / 0.3228` |
| test HR@10 / NDCG@10 | `0.4640 / 0.2835` |

相对 global soft，valid/test NDCG 从 `0.2633/0.2301` 提高到 `0.3228/0.2835`；相对 active-prefix，test NDCG 从 `0.2353` 提高到 `0.2835`。

#### 结论边界：hard-ST 对照能够证明什么

hard-ST 与官方基线并不是只差一个布尔开关：官方 RQ 预算为 400k，hard-ST 主矩阵为 100k；hard-ST 还使用全局和前缀正则及对应参数。因此它能支持“整套 hard-ST 训练方案更好”，不能把全部增益精确归因于 straight-through hard assignment 单项。

更干净的证据来自 soft 与 hard 的短探针对照：相近预算和正则框架下，对齐 hard bucket 后，真实 usage、unique 和 H(full) 立即大幅改善。完整下游的组件级因果仍需要专门消融。

## 同样的容量乘积，为什么不同排列不等价

三层容量分别为 $K_0,K_1,K_2$。这组实验固定

$$
K_0K_1K_2=2^{24}=16{,}777{,}216,
$$

只改变容量放在前层、中层还是后层。这样能排除“某组只是纸面组合空间更大”，但不能排除不同输出头和量化几何带来的端到端差异。

#### 结论边界：容量排列同时改变了哪些因素

改变 $K_l$ 至少改变三件事：

1. 第 $l$ 层 quantizer 可用的 Voronoi 区域数量；
2. 后续 residual 分布和 prefix group 的样本数；
3. decoder 在第 $l$ 个位置的分类头大小及标签频率。

因此这组实验识别的是端到端容量配置效应。若要单独估计“tokenizer 结构效应”，需要固定 SID 后改 decoder；若要单独估计“decoder label-space 效应”，需要构造受控的映射或蒸馏实验。

#### 技术细节：唯一 ID、singleton、最大桶和覆盖率

设实际出现过的完整 SID 集合为 $\mathcal S_{\mathrm{obs}}$，每个完整 SID 桶内有 $n_s$ 个 item。

unique SID 比例为

$$
r_{\mathrm{unique}}=\frac{|\mathcal S_{\mathrm{obs}}|}{N}.
$$

singleton item 比例为

$$
r_{\mathrm{singleton}}
=\frac{1}{N}\sum_{s\in\mathcal S_{\mathrm{obs}}}\mathbf 1[n_s=1].
$$

最大完整桶为

$$
n_{\max}=\max_{s\in\mathcal S_{\mathrm{obs}}}n_s.
$$

本文表格中的“有效路径覆盖率”是

$$
r_{\mathrm{eff}}=\frac{N_{\mathrm{eff}}}{N},
$$

它不是 unique SID 比例。unique 只数非空桶；$N_{\mathrm{eff}}$ 还会惩罚桶大小不均匀。

## Musical：条件有效分支过快收缩是不是原因

Musical 不是整体表现差。它的最佳 valid NDCG `0.3882` 高于 Sports、Toys 和 Office。所谓“反例”，只指容量排列的相对排序与多数数据集不同。

#### 查看完整数据：Musical 三个关键配置

| Musical 配置 | valid NDCG | RQ reconstruction | 有效分支 $B_0/B_1/B_2$ | H(full) | 有效路径 | unique SID |
|---|---:|---:|---:|---:|---:|---:|
| 后置 `[128,256,512]` | **`0.3882`** | **`0.1005`** | `99.4/36.25/2.28` | `9.0156` | `8,231` | `93.18%` |
| 中度前置 `[512,256,128]` | `0.3698` | `0.1068` | `488.6/10.44/1.61` | `9.0127` | `8,207` | `93.02%` |
| 极端前置 `[1024,256,64]` | `0.3630` | `0.1082` | `970.3/6.56/1.34` | `9.0527` | `8,541` | **`95.50%`** |

极端前置在有效路径和 unique 上都更高，却在 validation NDCG 上最低。这是“总路径越多越好”最清楚的反例。

#### 技术细节：总熵相近时，层间分配仍可完全不同

后置配置的熵链分解为

$$
H_{\mathrm{back}}
=4.598925+3.590402+0.826317
=9.015643.
$$

极端前置为

$$
H_{\mathrm{extreme}}
=6.877582+1.880629+0.294480
=9.052691.
$$

定义后两层信息占比

$$
r_{\mathrm{later}}
=\frac{H(S_1\mid S_0)+H(S_2\mid S_0,S_1)}
{H(S_0,S_1,S_2)}.
$$

代入得到

$$
r_{\mathrm{later}}^{\mathrm{back}}
\approx\frac{3.590402+0.826317}{9.015643}
\approx 49.0\%,
$$

而

$$
r_{\mathrm{later}}^{\mathrm{extreme}}
\approx\frac{1.880629+0.294480}{9.052691}
\approx 24.0\%.
$$

两组总信息量接近，但极端前置把大部分信息压到第一层，后两层条件增量迅速耗尽。它可以生成更多 unique SID，却不再形成同样强的“粗分后继续细化”结构。

#### 证据解释：生成模型的分层损失说明了什么

| Musical 配置 | hierarchy-0 loss | hierarchy-1 loss | hierarchy-2 loss | total loss |
|---|---:|---:|---:|---:|
| 后置 | `3.747` | `2.782` | `0.800` | `7.3296` |
| 等宽 | `4.433` | `2.187` | `0.704` | `7.3246` |
| 中度前置 | `4.989` | `1.742` | `0.506` | **`7.2363`** |
| 极端前置 | `5.465` | `1.390` | `0.427` | `7.2830` |

容量越前置，L0 分类越难，L1/L2 越容易。这与“后层条件分支枯竭”一致。但总 loss 最低的是中度前置，NDCG winner 却是后置，因此 token-level likelihood 也不是排序质量的充分代理。

#### 结论边界：数据量还不是 Musical 反例的充分解释

Musical 确实最小：150,121 条 train interactions、9,137 个 item。但现有跨数据集对照无法把规模确认为唯一原因：

- Toys 的交互/item 为 `22.36`，与 Musical 的 `21.63` 接近，却明显偏好中度前置。
- Video 的总交互量约为 Musical 两倍，偏好极端前置。
- Office 的 item 更少于 Sports/Toys，但中度前置仍略胜。

更直接的训练支持度诊断是每个活跃 L0 code 的样本中位数：

| 数据集 | 后置 | 等宽 | 中度前置 | 极端前置 |
|---|---:|---:|---:|---:|
| Musical | `1,073` | `443` | `211` | `97` |
| Video | `2,260` | `1,048` | `468` | `220` |

极端前置会把 Musical 的每个 L0 类支持度压得更低，这可能增加早期路由估计方差。但这些统计是观察到结果后的诊断，不是预注册的因果检验。

真正检验规模效应，需要把 Video 下采样到 Musical 的 train interactions，同时保持 Video 的 item/metadata 结构，再重复容量矩阵。

#### 补充线索：商品类目层级透露了什么

我们把 Amazon metadata 类目树当作一个后验代理，观察目录粗层是否集中、深层是否继续展开。这不是训练标签，也不是证明，只是帮助提出机制假设。

| 数据集 | metadata 粗层有效类目 | 最大粗类目质量 | 同类转移率 |
|---|---:|---:|---:|
| Sports | `2.45` | `61.3%` | `0.694` |
| Toys | `17.33` | `11.0%` | `0.342` |
| Office | `1.75` | `84.3%` | `0.795` |
| Video | `9.96` | `19.4%` | `0.543` |

Video 的 metadata 在粗层已经较分散，极端前置可以直接编码更多粗分辨率；Musical 粗层更集中，但深层路径从 `4.83` 扩展到 `12.63`，再扩展到 `63.28`，更像“少数粗类下面继续细分”。这个形状与 Musical 偏好后置一致，但还不能排除文本几何、行为序列和 decoder 优化的共同作用。

### 当前最严谨的表述

现有证据支持下面这个有限命题：

> Musical 中，容量越极端前置，后两层条件有效分支越快收缩；这种信息分配与较差 NDCG 同时出现，因此它是一个有数据支持的原因候选。

现有实验没有证明它是唯一原因、充分原因或任意数据集上的普适规律。

## 哪些结论已经站住，哪些仍只是解释

#### 证据分级：当前五个数据集稳定支持的结论

| 命题 | 直接证据 |
|---|---|
| 官方 TIGER 的早层坍缩跨数据集出现 | 五数据集官方 L0 有效分支只有 `1.00-6.19` |
| usage 不能单独判断表示质量 | layerwise+reset 后层 usage 100%，valid NDCG 仅 `0.1433` |
| H(full)、unique、重构损失不能单独选容量 | Sports 与 Musical 的极端前置反例；后置低 recon 不总是最好 |
| coarse-to-fine 顺序有任务价值 | 固定 SID 反序在 5/5 数据集 validation 和 test 都下降 |
| 容量分配具有任务依赖性 | 3 个中度前置 winner、1 个极端前置、1 个后置 |
| tuple 指标受碰撞乐观偏差影响 | 三数据集出现 item 更差但 tuple 更高 |

#### 证据分级：只得到部分支持的命题

| 命题 | 当前判断 | 缺少的证据 |
|---|---|---|
| `[512,256,128]` 是普适最优 | 不成立为普适命题，只是 3/5 winner、平均排名最好 | 非 Amazon、更多 seed、不同 decoder |
| 层间条件分支平衡比总路径更重要 | 有强关联证据 | 固定总熵并单独操控条件分支 |
| Musical 由后层分支衰减过快导致 | 合理机制候选 | oracle-prefix、层损失和 prefix-entropy 干预 |
| 小数据更偏好后置容量 | 现有结果不足 | 匹配领域结构的下采样实验 |

#### 证据分级：已经被削弱的简单规则

- 让更多 code 被使用，推荐自然提高。
- unique SID 或有效路径越多，NDCG 必然越高。
- RQ reconstruction 越低，下游必然越好。
- decoder 训练更久就能补回 tokenizer 的结构缺陷。
- 三层 codebook 只看乘积，排列和顺序无关。
- 所有数据集都应该把容量尽可能前置。

## 下一步怎样把 Musical 的解释变成因果证据

下一轮不应继续无目的扩大容量网格。最有信息量的是固定尽可能多的部分，只干预一个机制。

#### 后续验证：六个按优先级排列的最小实验

1. **Oracle-prefix decoder，不重训 RQ。** 推理时先强制正确 $S_0$，再强制正确 $(S_0,S_1)$。若极端前置在给定正确 prefix 后迅速追平，主要瓶颈是早期路由错误传播。
2. **固定 SID，只改 L0 loss weight 或 label smoothing。** 如果极端前置恢复，说明宽 L0 的 decoder 可学性贡献较大，而不是 tokenizer 信息量本身不足。
3. **固定容量，提高 prefix-entropy 权重。** 预注册 `.002 -> .005/.01`，检查 $B_1/B_2$ 与 NDCG 是否同步上升。
4. **补同总容量后置组。** `[128,512,256]` 与 `[64,512,512]` 用于区分“首层小”与“中层需要更强展开”。
5. **Video 下采样到 Musical 规模。** 单独操控样本量，检验“小数据导致后置偏好”。
6. **多 seed 与非 Amazon 复现。** 在主配置冻结后运行，而不是根据 test 结果临时追加。

这六项都属于后续验证，不进入本文结果。尤其是多随机种子实验，应在主配置冻结后统一运行，不能根据当前测试集表现临时决定追加哪些配置。

### 本文没有声称完成模型级 SOTA 比较

这篇文章比较的是 TIGER 内部的训练和容量设计，不是一个覆盖所有生成式推荐方法的 SOTA 榜单。LETTER、ETEGRec、MME-SID 等 recommendation-aware tokenizer 尚未按同一数据划分和评价协议复现，因此本文不会把它们写成已经完成的胜负结论。

## 实验怎样保证比较公平

数据来自公开的 [Amazon Review Data 2018](https://cseweb.ucsd.edu/~jmcauley/datasets/amazon_v2/index.html)。我们使用 Sports & Outdoors、Toys & Games、Office Products、Video Games 和 Musical Instruments 五个类目，并保持各组实验的数据预处理、候选集合和下游模型框架一致。

为了避免“看完测试集再挑模型”，实验遵循以下规则：

1. **只用验证集选择 checkpoint 和容量配置。** 测试集、动态测试和 tuple-test 只用于最终报告。
2. **容量排列实验固定总组合容量。** `[128,256,512]`、`[256,256,256]`、`[512,256,128]` 和 `[1024,256,64]` 的乘积都等于 $2^{24}$。
3. **顺序消融固定同一份语义 ID。** 正序与反序只改变生成模型看到 token 的顺序，不重新训练量化器。
4. **商品级指标是主结论依据。** tuple 指标把命中同一语义 ID 也算作成功，存在碰撞带来的乐观偏差，因此只作诊断。
5. **所有主矩阵使用同一个随机种子。** 这保证了当前矩阵内部可比，但也构成本文最重要的统计局限。

#### 方法边界：哪些差异不能被归因给单个开关

官方基线的 RQ-VAE 训练预算为 400k step，hard-ST 主矩阵为 100k step；hard-ST 还同时使用了全局和前缀分布正则。因此，本文能够支持“整套 hard-ST 训练方案优于未改动的官方配置”，但不能把全部增益精确归因于 straight-through hard assignment 这一个改动。

此外，主矩阵目前只有 seed2024，且全部来自 Amazon 类目。跨随机种子稳定性、非 Amazon 数据集复现，以及 Musical 条件分支假设的因果验证仍然是后续工作。

## 结语：不要再问“码本有没有用满”，先问“信息为什么要放在这里”

这组实验最初像是在修复一个工程故障：256 个码字为什么只剩 1 个。走到最后，它变成了一个更有研究价值的问题：一个层级离散表示怎样分配早期路由、条件展开和末端消歧，才与生成式推荐的预测顺序和数据结构匹配。

hard-ST 给出了一个可靠的训练起点；中度前置给出了一个稳健但非普适的默认配置；Musical 则提醒我们，总熵、唯一 ID 比例和有效路径数都不能替代层间信息结构。下一步真正值得做的，不是再堆一个更大的码本，而是用受控实验确认：**哪些数据结构需要早分，哪些需要晚分，生成模型的错误又是从路径的哪一层开始放大。**
